Client Project Overview

KBT Reading Support - Project Overview

This application will serve my mother's tutoring business for kids with dyslexia. The purpose of this application is to serve as a portal for the owner as well as the parents of the students. The intended users of this website are the owner and the parents, not the children.

The content of the website will include a calendar of a parent or the owner's sessions for that month. Additionally, there will be options to send or view an invoice, cancel or reschedule one or all sessions, write and view session notes, email the owner or email a parent, join the Zoom session, and add or remove a client. The parents and the owner will have a different dashboard.

Page-by-page Design

  1. Name: Registration Page
  2. Purpose: For parents who receive an invite email to register as a user, schedule their sessions, and receive login credentials.
  3. Intended Users: Parents/Clients

Dynamic Functionality

To begin, there will be forms across the website. Namely, forms on the cancel and reschedule page, on the message page, invoice page, session note page, etc. These forms will have interactive dropdowns that filter data and display them as options, such as showing the sessions for one specific client on the session notes page. This functionality would be something similar to the form page of this website. The purpose of this functionality is to consolidate the collection of data from the user in an organized manner.

Additionally, both dashboards will have an interactive calendar, where one can click on a day that contains a session and see information about that session in a pop-up window. This is something that will be similar to Google Calendar. There will be automated emails that come with some of the operations, such as on the invoice page, message and notes pages, etc. Clients who are invited will receive an invitation email that will lead them to the registration page. Data will be dynamically extracted and put into the email. There will also be session reminder emails. An example of a site that uses automated emails heavily is Canvas. The purpose of this is to display the schedule of the user in a more suitable way than just text.

Most of the functionality and interactivity will be in the form of database operations. For example, the database will retrieve only the time slots that are available in the registration and reschedule pages. The database will include clients and their information, scheduled sessions, and messages as well as session notes. This data will be dynamically adjusted in the cancel/reschedule page, message and session note pages, etc. There are a plethora of websites that work with a database, such as the Canvas site mentioned above. The purpose of the database is it is necessary in order to keep track of the things the site owner needs to keep track of.

I will most likely add interactive images to the about page. This could be either in the form of a slideshow, an image that changes when you hover over it, an image that moves around, etc. An example of a site that has images with multiple types of interactivity is tonyrobins.com. The purpose of this is to make the website more engaging and appealing.
